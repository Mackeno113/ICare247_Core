⚡ Bước 1: Chuẩn Bị Project

Trước khi dùng bất kỳ prompt nào, thực hiện 2 việc sau:

1

Copy CLAUDE.md vào project root

File CLAUDE.md là "bộ nhớ" của Claude Code cho project này. Mỗi khi bắt đầu session mới, Claude Code tự động đọc file này và hiểu ngay toàn bộ tech stack, convention, DB schema, mà không cần bạn giải thích lại.

2

Tạo solution structure

Đảm bảo đã có đủ các project trong solution: ICare247.ConfigStudio, ICare247.ConfigStudio.Core, ICare247.ConfigStudio.Modules.Forms, .Modules.Rules, .Modules.Events, .Modules.Grammar. Claude Code sẽ tạo files vào đúng project nhờ CLAUDE.md.

💬 Bước 3: Cách Dùng Từng Prompt

Cách 1 — Copy toàn bộ nội dung trong block ```...```

Mở file prompt, copy phần text trong block ```...```, dán thẳng vào Claude Code chat.

💡 Mẹo: Mở file prompt ngay trong Claude Code

Bạn có thể nói với Claude Code: "Đọc file prompts/01_shell_navigation.md và thực hiện theo đó" — Claude Code sẽ tự đọc file và thực thi mà không cần copy-paste.

Cách 2 — Yêu cầu đọc file (khuyến khích)

# Gõ vào Claude Code chat: Đọc file prompts/01_shell_navigation.md và implement theo đúng spec trong đó. Nhớ đọc CLAUDE.md trước để nắm convention.

Cách 3 — Làm từng phần nhỏ (tốt nhất cho màn hình phức tạp)

# Ví dụ cho Prompt 05 (Val Rule Editor — phức tạp): # Bước 5a: Chỉ làm layout XAML trước Đọc prompts/05_validation_rule_editor.md. Chỉ tạo file ValidationRuleEditorView.xaml với đầy đủ layout. Chưa cần ViewModel — dùng mock data trong code-behind để preview. # Bước 5b: Sau khi xem layout OK, làm ViewModel Tiếp tục với ValidationRuleEditorViewModel.cs theo spec trong file prompt. Implement Quick Config cho Numeric và Regex trước. # Bước 5c: Thêm Expression integration Implement kết nối với ExpressionBuilderDialog (đã tạo ở prompt 07). Thêm AutoGenerateExpression logic cho tất cả rule types.

🎯 Bước 4: Tips Để Prompt Hiệu Quả

✓

Luôn bắt đầu bằng "Đọc CLAUDE.md"

Câu này đảm bảo Claude Code load toàn bộ context project trước khi code. Nếu không có câu này, Claude Code có thể dùng sai naming convention hoặc pattern.

✓

Commit sau mỗi prompt

Mỗi prompt = 1 feature nhỏ hoàn chỉnh. git commit sau khi screen chạy được → dễ rollback nếu prompt tiếp theo có vấn đề. Không nên gộp nhiều prompt vào 1 commit.

✓

Yêu cầu mock data cụ thể

Các prompt đã có sẵn mock data spec. Đảm bảo Claude Code tạo mock data đó để screen có nội dung hiển thị ngay — không bị trắng blank.

✓

Kiểm tra build trước khi tiếp

Sau mỗi prompt, yêu cầu: "Build solution và fix tất cả compile error trước khi tiếp tục". Nếu có lỗi, Claude Code sẽ tự sửa trong cùng session.

⚠️ Thứ tự Prompt 05 → 07

Prompt 05 (Rule Editor) mở ExpressionBuilderDialog nhưng Prompt 07 (Dialog) chưa được tạo. Giải pháp: Khi làm Prompt 05, yêu cầu Claude Code tạo interface/stub cho ExpressionBuilderDialog trước, sau đó làm Prompt 07 implement đầy đủ. Hoặc làm 07 trước 05.
